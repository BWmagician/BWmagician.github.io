623E. Transforming Sequence

A very good problem. I remembered that I spent a night and a morning to solve this problem.

Fistlly we should realize the only thing matter is if we can put a new bit in the position to make the XOR sum sequence strictly increasing. If we just want to find a normal recurrence formula, we can set $f[i][j]$ to be “counted til $i$, and using $j$ bits”. But we are numerating the information about a two dimension thing, which is a huge consumption as at least $O(n^2)$.

So there is a need to increasing the information density a little bit. The same as saying $f[i][j]$ has some repeated or meaningless information in it. You will find the most troublesome thing in this problem is to calculate the contribution due to the free choice of all the selected bits in previous positions. For example, if you have chosen $k_0$ bits in your previous positions. you can use them freely, so you have to multiply addtional $2^{k_0}$ to your contribution. Regardless this part, the problem talks only about grouping, which is a pure combination stuff.

So we can try to solve this problem in combinactorial method. Try to find the relationship between $f_{a+b}$ and $f_a,f_b$, where $f_i[j]$ stands for the contribution of using $j$ bits in sequence of length of $i$.

The formula will look like this, but we haven’t define what does $f$ means here.

Here $r$ is some coefficient but we can just consider it later. This formula simply tells, if we trying to attach a sequence of length $a$ in fornt of a sequence of length $b$, it should count $2^j$ free choice for every number in $b$ sequence to make every bits in the $a$ sequence contributed. Easy to prove every $f_{a+b}$ could be counted in this way. But here is the thing, how to make sure every count are unique? That’s the same as to decide what does $f$ means and what is $r$ here.

Let’s first try if $f_{n}[i]$ could stand for the real counting of using exactly $i$ bits in a sequence of length of $n$. Then here is the question, we can’t decide $r$. Notice that $f_a$ is the exact counting while $f_b$ is the exact count, we can’t distinguish if they use the same bit in the $k$ bits given by the problem. So this would not work. If it work, the coefficient $r$ would be even more complicated to calculate.

A smart method is to avoid distributing the real bits at very beginning, we can just let $fn[i]$ be counting for unlabelled bits. This means at very beginning we regard every bit to be the same. And then when we compute $f{a+b}[i]$, we can just regard $r=1$. Because if we regard every bit to be the same, we are just doing pure grouping, there is no additional coefficient!

At last, we assign different bits to our grouping method to make it to be the real coutning. However, in this problem, every elements in one group (your selected new bits for a number, their order doesn’t change anything) are regard as the same, so you should dedupe for every group you divided. We can have a case to see how can we do dedupe. Practically, if you divide $j$ bits into $j_0,j_1,j_2,j_3$, four group, you actually have to multiply $\displaystyle \frac{(_j ^k)j!}{j_0!j_1!j_2!j_3!}$ to calculate its real counting, the denominator is used to dedupe in a group. Since the denominator only relates to the size of each group, we can just multiply $\frac{1}{j_0},\frac{1}{j_1},…$ into $f_1$, and every calculation based on $f_1$ would dedupe based on the groups’ sizes automatically. So the real counting for a sequence of length $n$ using $j$ bits would be $(_j ^k)j!f_n[j]$.

Here is the full procedure of this problem, we just let $f_n[i]$ be $[x^i]F(x)$, this means $f_n[i]$ is the coefficient of term $x^i$ in $F(x)$.

Actually what we did to dedupe is similar to EGF (expotential generating function). I realize this thing after I learned how to compute stirling number of the first kind. Saddly when I trying to solve this problem, I had no idea of what expotential function can do. 力竭了
