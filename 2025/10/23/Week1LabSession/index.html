  2. Add the path to Environment Variable. This simply means when you type in java and javac in your terminal, the terminal will refers to the java.exe and javac.exe by the path you provided.

2’s Complementary Code

32 31 30 … 3 2 1
$-2^{31}$ $2^{30}$ $2^{29}$ … $2^2$ $2^1$ $2^0$

$(111…111)_2=-1$
$(111…110)_2=-2$

There is 1 bit overflows(The overflow bit is useful to check whether you have the valid result, but it will not presented in the result), the result will only be $(000…000)_2$

This simply means that $(1)_2 + (-1)_2 = 0$

For finding the complementary code of $-100$, we write out the binary code of $100$ first.

Find the inversion $x$ of this binary code (Inversion simply means turn $0$ to $1$, turn $1$ to $0$ for each bit).

Then, $x + 100$ is,

This means that $(x)_2+(100)_2=(-1)_2$

We know that $(-100)_2+(100)_2=0$

Thus, (Notice, we can only use addition now, because we don’t define substraction yet..)

So, $(-100)_2=(111…1110011011)_2+1=(111…1110011100)_2$

For every valid positive number $x$, we can say ,in computer, $(-x)_2=inverse(x)+1$
