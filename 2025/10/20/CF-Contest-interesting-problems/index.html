2154E. No Mind To Think

https://codeforces.com/contest/2154/problem/E

Notice that after choosing one value to be the middle, the optimal choice for the next step is still that value.. In other word, we should sort all the value and choose an interval to start with. The following steps are using the chosen value to increase the samllest values.

Here comes the monotone freatures. Once you fix a middle value, the longer the interval you choose, the lesser benefit you will get. Because you will decrease a larger number but increase also larger values after the previous step. This will make the contribution by length of the interval look like convex at the position of a middle value.

1004F. Sonya and Bitwise OR

Our first thought about interval OR sum should all be the same —-It only increase when the range get larger. Thus, if we ingore the modify opereation, we can directly have certain intervals $[i,i+j],i\in[1,n]$, so that the OR sum of $[i,i+j]$ is at least $x$ and $j$ can not be smaller. These kind of intervals should not exceeds the number of $n$. Given $[l,r]$, we only need to determine how many sub-interval of $[l,r]$ includes the listed intervals. The listed intervals, due to the OR sum, have kind features that $i+j$ always increases when $i$ get larger. When calculating contribution by $[i,i+j]$, we only need to count $r-(i+j)+1$ intervals which has the left border of $i$..

But how about the modifying operation. It forces us to change our way to think. The idea above actually calculate the contribution through point-interval method. To modify certain point, it will have a huge cost of modifying contribtion from point to interval. But what about interval-interval contribution? This drives us to think about segment tree (or DAQ method), we find meaningful value of an interval are, sum of the interval, sum of sub-intervals touching left border and sum of sub-intervals touching right border. You may think thera are a big amount of sub-intervals touching the border.. Yes, but do you find most of them may have the same value? Due to OR sum, the OR sum of sub-interval touching border will be increasing only when the length get larger.. So that means it will at most has $\log a$ different values for all the sub-interval touching the same border.. Then we can solve this problem.
