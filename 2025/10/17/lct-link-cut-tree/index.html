lct,link-cut tree

Created2025-10-17|Updated2025-10-18
|Post Views:

Time to review the passed problems.

template

https://www.luogu.com.cn/problem/P2387
Maintain the xor sum of a link.

Tips:

  1. Notice the dashed link has its upper end refers to the real node but its lower end refers to the most left node in the link. So, when doing the rotation, don’t add children to the father of the dashed link.
  2. Good habit to set rev as a tag after doing the operation on itself.
  3. The pushdown and pushup in the splay should not be removed for the case of only root in a link. If the rev remains when doing the access, it will be fatal.
  4. Xor is okay, if the tag you maintaining is related to location in the link, the reverse should have a additional contribution to that tag.

Classical problem

https://www.luogu.com.cn/problem/P2387
Since the problem only asks us to maintain the maximum value.
We can first consider sort edges by $a$, then whenever the upperbound of $a$ is chosen, the previous edge can be used freely.
Then we consider $b$, through these edges, we need to find the lowest $b$ to connect $1$ to $n$.
We can apply lct to do this. Every time we add a new edge, consider if a cycle will exist. If no, add the edge. If yes, consider exclude the maximum value of the cycle. This means, if $b$ of the new edge is lower than the maximum $b$ of the cycle, we can add this edge and pop another to maintain the shape of the tree.
