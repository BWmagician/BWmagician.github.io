CornerPad

Strictly speaking, the worst case of this code achieves $O(2^n)$ by ,for example, making the $d_n$ be $R$ and make all other $d_i$ to be 1. In this specific case, you won’t get the right anwser before you pick the last item, which force you to exame all the combination if you starts from left side. However, in most case, practically speaking, the code runs efficiently.
Here we only need to pick a combination, so no need to check starting from $1$ to $n$. We give it an order to always pick items from left side to right side which cut all the repeated situations.
