Big O Notation

Big O is a asymptotic analysis of the run time of an algorithm.

Concept in Math

This statement simply means when n is large, $f(n)$ is at the same level of $g(n)$ or even higher level of $g(n)$. And the equivalence here means $g(n)$ is in the set of $O(f(n))$. You can regard $O(f(n))$ as a set that includes all the functions of the same (and lower) level of $f(n)$.

Actually the statement should be like $g(n) \in O(f(n))$ if we only focus on the meaning.

PS: Please always try to find a tight upper bound in homework and tests. Otherwise, your analysis of the runtime is not accurate!

Important things

  1. A piece of code in $O(1)$ can be slower than another piece of code in $O(n)$. Big O tells nothing if $n$ is small.
  2. When $n$ is large, we can say code in $O(1)$ is more efficient than $O(n)$.

Array

Array is a kind of data structre that supports rondom access. Because an array always occupies continuous addresses in the memory which naturally fits the idea of random access(meaning that it only takes $O(1)$ to get, for exapmle, the value of $a[100000]$).

Singly Linked List and Doubly Linked List

There is no big difference. But doubly linked list is more efficient if you want the information of what is the previous node.

For singly linked list, for a given node, if you want to find the previous node, you have to start from the head to search which is $O(n)$. But for doubly linked list, if you simply want to access the previous node, you can directly do that.

You should also pay attention to the insertion in doubly linked list. To implement, you have to match four new edges, but paying attention to where you get these information. We say you want to insert new node $x$ between $u$ and $v$. We say we are now at $u$,

The order 1 2 3 4 is not correct,
Once 1 is implemented, $u.next$ will change and you can’t do others.
One valid order is 2 3 4 1. You can also use other method to match edges, this is just an reminder of taking care of your method.

Stack and Queue

Queue

First in, First out.

Example: badminton shuttlecock tube. The badminton you get from side is always the first one you put in another side.

Template Problem: Caterpillar

Regard the head of the caterpillar as tail of a queue.
Regard the tail of the caterpillar as head of a queue.
It is first-in-first-out. Becase the caterpillar only moves to the right and never go back, the first position head touches will be the first position tail leaves.
Then head moving is pushing value into the queue.
Then tail moving is poping value out of the queue.

When counting the color, we do a count for every color. The number of differnt color increases when a new color appears. The number of differnt color decreases when a kind of color disapears (see the code).

Stack

First in, last out. (Last in, first out)

Example: The recursion. The computer processes the most recently called function first(Last in, first out. You can regard the most recent called method is at the top of a stack).

Template Problem: StackQuestion

We can simulate the process by following the push order. Only when the top of the stack is the same as the first value in remaining pop order, we can do a pop. Since each time we either do a push or a pop, if we don’t choose to do a pop when the top of the stack meets the pop order, we will never have a chance to pop this value because the next pushed value will become the top and the pop order will be stuck at present first value.
