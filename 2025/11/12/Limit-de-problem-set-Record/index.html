CF718C Sasha and Array

The idea is simple. For $fa+f_b+f_c$ store the next one $f{a+1}+f{b+1}+f{c+1}$ which can help to do recurrence.

Reducing the constant is difficult

P1712 [NOI2016] 区间 - 洛谷

Obviously two ways.

The first is fixing a point try to find the minimum range of continuous segments… which seems difficult, because when you move the point to next position the length of segment disappears and adds randomly.

The second is finding whether an answer satisfied in an order of length. Once following an order of length, the best choice is to choose all the segments in a range of lengths. Then we can do something like a queue operation to find every minimum satisfied solution for every maximum length.

P3792 由乃与大母神原型和偶像崇拜 - 洛谷

Notice that continuous number in a range is equivalent to 1. min-max==r-l 2. no repetitive number.

Do something like color counting (record the previous positions, determine the repeating by checking whether there are “previous positions” in the present range. If no , no repeating)

Also can use hash map.. Define own rule of checking whether the numbers are continuous. The more rules you make the more precise you are. Since m queries, it’s better to make expectation for getting wrong below $O(\frac{1}{m^2})$
