如果不装这个插件会产生很奇怪的bug

更换渲染插件

由于hexo自带的渲染器hexo-renderer和markdown语法有很多冲突。为避免页面出现布局混乱，需要更换一个渲染器。

卸载默认渲染器

$ npm uninstall hexo-renderer-marked

装hexo-renderer-markdown-it

$ npm install hexo-renderer-markdown-it

数学公式

bw主题已经配置了mathjax的cdn

由于hexo-renderer-markdown-it的基础渲染器markdown-it在文章渲染时设置了转义，所以为了打出一个\,你必须要再打一个\, 比如说为了打出{，你要写$\\{$。
这个只需要在转义的代码里把你想要保留的符号删掉就行，代价是你在正文中无法使用这些符号的转义，看起来没什么影响。

当你安装了hexo-renderer-markdown-it后，在Root\node_modules\markdown-it\lib\rules_inline\escape.js中修改，具体来说，把第11行的代码里面的花括号和下划线删了就行。

2025/6/30

分类

更改了之前的差分逻辑。

以前的逻辑是给每一个子页面都搞一个html，这个复杂度其实是有问题的。

现在改为url传参，让页面可是实时更改差分。好处可以把所有子页面都闪光了，只在分类页面做修改。那么有一个问题，在js代码工作的时候，hexo的搭建已经结束了，也就是说我们用不了helper函数。所以需要创建一个json文件来存放处理好的数据，让js每次都读这个数据文件就行。看起来复杂度是正确的呢（笑。

需要注意的是，分类路径的名称不可以出现最前面和最后面的空格。比如" uuz"和"uuz "是不合法的，而"uuz"是合法的。原因是hexo对category的处理会自动砍掉这两个空格，未知如何修改，所以保持路径合法即可（没有人会需要这两种空格吧。
