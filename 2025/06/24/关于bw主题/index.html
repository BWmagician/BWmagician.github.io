下载

下载主题

$ git clone https://github.com/BWmagician/hexo-theme-bw.git themes/bw

下载图片插件

$ npm install https://github.com/CodeFalling/hexo-asset-image --save

如果不装这个插件会产生很奇怪的bug

更换渲染插件

由于hexo自带的渲染器hexo-renderer和markdown语法有很多冲突。为避免页面出现布局混乱，需要更换一个渲染器。

卸载默认渲染器

$ npm uninstall hexo-renderer-marked --save

装hexo-renderer-kramed

$ npm install hexo-renderer-kramed --save

数学公式

bw主题已经配置了mathjax的cdn

由于hexo-renderer-markdown-it的基础渲染器markdown-it在文章渲染时设置了转义，所以为了打出一个\,你必须要再打一个\, 比如说为了打出{，你要写。

这个只需要在转义的代码里把你想要保留的符号删掉就行，代价是你在正文中无法使用这些符号的转义，看起来没什么影响。

当你安装了hexo-renderer-markdown-it后，在Root\node_modules\markdown-it\lib\rules_inline\escape.js中修改，具体来说，把第11行的代码里面的花括号和下划线删了就行

2025/6/30

分类

更改了之前的差分逻辑。

以前的逻辑是给每一个子页面都搞一个html，这个复杂度其实是有问题的。

现在改为url传参，让页面可是实时更改差分。好处可以把所有子页面都闪光了，只在分类页面做修改。那么有一个问题，在js代码工作的时候，hexo的搭建已经结束了，也就是说我们用不了helper函数。所以需要创建一个json文件来存放处理好的数据，让js每次都读这个数据文件就行。看起来复杂度是正确的呢（笑。

需要注意的是，分类路径的名称不可以出现最前面和最后面的空格。比如” uuz”和”uuz “是不合法的，而”uuz”是合法的。原因是hexo对category的处理会自动砍掉这两个空格，未知如何修改，所以保持路径合法即可（没有人会需要这两种空格吧。

2025/7/25

大更新

布局改变

为了适配手机端，索性直接把navigation放到了上面，删掉首页的神秘背影图

手机端适配

碰到了很多问题，最神奇的是苹果手机的浏览器自动把小字体加大。这个问题导致文字和markdown渲染出来的字体大小不一致，非常难想到。

大部分的布局都改成了居中，某些图标和字体做了手机端的差分修改。

具体有：
1.手机端强制隐藏asider
2.手机端代码块强制不可以换行，电脑端还是有缩放换行的功能的，虽然有点意义不明
3.手机端的各种字体都有适当比例的放大
4.手机端的各种padding和margin也都有微调
5.迷你八卦炉的图标也微调了

亮暗模式

为了调亮暗模式，发现css有一个很好用的:root属性，感觉就是css的内置变量，能直接用js改，非常方便。

虽然亮暗模式点击意义不明的迷你八卦炉看起来缺乏引导，但是还是就先这样吧。

顺便知道了还有一个localstorage的功能，看起来能比cookie存更多东西，这个确实方便，以后页面间互动也不用在url里加?了。
